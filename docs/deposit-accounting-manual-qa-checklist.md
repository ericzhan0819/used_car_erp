# 訂金保留到會計入帳手動 QA 清單

## 目的

固定目前已完成的中古車訂金流程，確認業務端只建立金流紀錄與傳票草稿，會計端再從「會計作業」工作區人工確認入帳。

本文件只驗證既有流程，不新增尾款、交車、出庫、銷售發票或收款單。

---

## 測試前準備

站台：

```text
erpnext.localhost
```

更新後請執行：

```bash
cd ~/frappe/frappe-bench
bench --site erpnext.localhost migrate
bench --site erpnext.localhost clear-cache
```

如果 ERPNext UI 已開啟，請重新整理；若仍看不到新工作區，請登出後重新登入。

## 本次 QA 範圍

要驗證的主流程：

```text
新增車輛
→ 完成入庫
→ 上架中
→ 建立訂金保留
→ 系統建立金流紀錄
→ 系統建立待審核傳票草稿
→ 會計作業
→ 待審核傳票草稿
→ 確認入帳
→ 建立正式會計傳票
```

本階段暫緩：

```text
尾款、貸款撥款、退款、正式出售、交車、出庫、銷售發票、收款單、完整收入認列、完整報表、租賃、完整權限系統。
```

- 不要修改 ERPNext 原生 `Accounting` 工作區。

### 會計作業

進入 `會計作業`，應看到：

```text
會計待辦

- 金流紀錄
```

| 入口 | 預期結果 |
| --- | --- |

| 傳票草稿 | 進入 `Used Car Voucher Draft` 列表 |

## 二、業務端訂金保留流程

### 1. 新增車輛

進入：

```text
中古車管理 → 新增車輛
```

填入必要資料並儲存：

- 廠牌
- 車型
- 年份
- 車牌
- VIN / 車身號
- 採購車價

### 2. 完成入庫

在車輛頁點擊 `完成入庫`。

預期結果：

- 車輛狀態變成 `庫存中`
- 系統建立 ERPNext 商品
- 系統建立並提交入庫單
- 系統建立 / 綁定序號
- 使用者不需要手動進 Item、Stock Entry、Serial No 操作

### 3. 上架車輛

依狀態點擊：

```text
直接上架
```

或：

```text
開始整備 → 整備完成並上架
```

預期結果：

- 車輛狀態變成 `上架中`
- 不新增額外入庫單
- 不更動正式庫存帳

### 4. 建立訂金保留

在 `上架中` 車輛建立訂金保留，填入：

- 客戶姓名
- 客戶電話
- 訂金金額
- 付款方式
- 付款日期
- 付款備註 / 末五碼，如有

預期結果：

- 車輛狀態變成 `保留中`
- 建立 `Used Car Reservation`
- 建立 `Used Car Money Flow`
- 建立 `Used Car Voucher Draft`
- 不建立正式 `Journal Entry`
- 不建立 `Payment Entry`
- 不建立 `Sales Invoice`
- 不建立 `Delivery Note`
- 不新增額外 `Stock Entry`

---

## 三、會計端確認入帳流程

### 1. 找到待審核傳票草稿

進入：

```text
會計作業 → 待審核傳票草稿
```

預期：

- 可以看到剛剛建立的訂金傳票草稿
- 草稿狀態為 `待審核`
- 列表可看到傳票日期、金流紀錄、車輛、借方合計、貸方合計與正式會計傳票欄位

### 2. 檢查草稿內容

點進傳票草稿，檢查：

- 草稿狀態是 `待審核`
- 金流紀錄連結正確
- 車輛連結正確
- 保留紀錄連結正確
- 客戶連結正確
- 借方合計等於貸方合計
- 借貸差額為 0

白話分錄：

```text
借：銀行存款 / 現金
貸：預收款 / 暫收款
```

意思是：

```text
公司收到訂金，但車還沒完成出售，所以先列預收款 / 暫收款，不直接算收入。
```

### 3. 確認入帳

在傳票草稿點擊 `確認入帳`，填入審核備註後確認。

預期結果：

- 傳票草稿狀態變成 `已入帳`
- 金流紀錄狀態變成 `已入帳`
- 建立正式 `Journal Entry`
- 正式 `Journal Entry` 已提交
- 傳票草稿上可看到正式會計傳票連結

---

## 四、禁止產生文件檢查

完成確認入帳後，確認沒有建立：

| 文件 | 預期 |
| --- | --- |
| Payment Entry / 收款單 | 不應建立 |
| Sales Invoice / 銷售發票 | 不應建立 |
| Delivery Note / 交車單 | 不應建立 |
| 額外 Stock Entry / 入庫單 | 不應因訂金或入帳再新增 |

目前正式入帳只應建立：

```text
Journal Entry / 正式會計傳票
```

---

## 五、退回與作廢檢查

### 待審核草稿可退回

建立一筆新的訂金保留後，進入對應傳票草稿，點擊 `退回草稿`。

預期：

- 草稿狀態變成 `已退回`
- 需要輸入退回原因
- 不建立正式會計傳票

### 待審核或已退回草稿可作廢

在待審核或已退回草稿點擊 `作廢草稿`。

預期：

- 草稿狀態變成 `已作廢`
- 金流紀錄狀態變成 `已作廢`
- 不建立正式會計傳票

### 已入帳草稿不可作廢

對已入帳草稿嘗試作廢，預期：

- 系統應阻止
- 不應直接刪正式會計傳票
- 未來若要反轉，應走反向傳票流程，不在本階段處理

## 七、通過標準

本輪 QA 通過條件：

- `會計作業` 工作區可見，且與 `中古車管理` 平級。
- `待審核傳票草稿` 可直接進入待審核列表。
- 建立訂金保留後，業務端不需要進會計頁操作。
- 建立訂金保留後，系統會建立金流紀錄與傳票草稿。
- 建立訂金保留後，不會直接建立正式會計傳票。
- 會計確認入帳後，才建立並提交正式會計傳票。
- 確認入帳後，不會建立收款單、銷售發票、交車單或額外入庫單。
- 驗證工具不留下孤兒金流紀錄或傳票草稿。
