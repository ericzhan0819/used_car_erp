# 尾款收款金流手動 QA 清單

## 目的

本文件驗證「尾款收款 / 成交前金流 Foundation」。

目前尾款只進入金流紀錄與傳票草稿，不代表已交車、已售出、已出庫或已正式認列收入。

---

## 本次流程

```text
保留中車輛
→ 建立尾款收款
→ 建立尾款金流紀錄
→ 建立尾款傳票草稿
→ 會計作業
→ 待審核傳票草稿
→ 確認入帳
→ 建立正式會計傳票
```

---

## 不做的事

```text
不交車
不出庫
不開銷售發票
不建立收款單
不把車改成已售出
不認列完整收入
```

---

## 一、前置條件

先完成既有訂金流程：

```text
新增車輛
→ 完成入庫
→ 上架中
→ 建立訂金保留
→ 車輛變成保留中
```

尾款建立前，保留紀錄應已有：

- 訂金金流紀錄
- 訂金傳票草稿

---

## 二、從車輛頁建立尾款

進入保留中車輛頁，應看到：

```text
建立尾款收款
取消保留
```

點擊 `建立尾款收款`，填入：

- 尾款金額
- 付款方式
- 尾款日期
- 付款備註 / 末五碼，如有
- 備註，如有

確認提示應明確說明：

```text
只建立金流紀錄與傳票草稿，不交車、不出庫、不開銷售發票、不建立收款單。
```

預期結果：

- 車輛仍為 `保留中`
- 建立 `Used Car Money Flow`
- 金流類型為 `尾款收款`
- 金流狀態為 `待審核`
- 建立 `Used Car Voucher Draft`
- 傳票草稿狀態為 `待審核`
- 保留紀錄寫入尾款金額、尾款日期、尾款付款方式、尾款金流紀錄、尾款傳票草稿

---

## 三、從保留紀錄建立尾款

進入有效的 `Used Car Reservation`。

若尚未建立尾款，應看到：

```text
建立尾款收款
```

點擊後流程與車輛頁相同。

若已建立尾款，應看到提示：

```text
此保留紀錄已建立尾款金流與傳票草稿，請到「會計作業」確認入帳。
```

---

## 四、會計確認尾款入帳

進入：

```text
會計作業 → 待審核傳票草稿
```

找到尾款傳票草稿，檢查：

- 摘要包含 `尾款收款`
- 金流紀錄連結正確
- 車輛連結正確
- 保留紀錄連結正確
- 借方合計等於貸方合計
- 借貸差額為 0

白話分錄仍為：

```text
借：銀行存款 / 現金
貸：預收款 / 暫收款
```

原因：車還沒有交付完成，目前仍先列預收款 / 暫收款，不直接算收入。

點擊 `確認入帳` 後，預期：

- 尾款傳票草稿狀態變成 `已入帳`
- 尾款金流紀錄狀態變成 `已入帳`
- 建立正式 `Journal Entry`
- 保留紀錄寫入 `尾款正式會計傳票`

---

## 五、禁止產生文件檢查

建立尾款與確認尾款入帳後，確認沒有新增：

| 文件 | 預期 |
| --- | --- |
| Payment Entry / 收款單 | 不應建立 |
| Sales Invoice / 銷售發票 | 不應建立 |
| Delivery Note / 交車單 | 不應建立 |
| 額外 Stock Entry / 入庫單 | 不應建立 |

車輛狀態仍應為：

```text
保留中
```

---

## 六、成交前檢查

成交前檢查只驗證訂金與尾款是否都已入帳，不會交車、出庫、開銷售發票或建立收款單。

通過流程：

```text
保留中車輛
→ 建立尾款
→ 會計確認訂金入帳
→ 會計確認尾款入帳
→ 按成交前檢查
→ 通過
```

預期通過訊息：

```text
此車輛已完成訂金與尾款入帳，可進入成交 / 交車流程。
```

不通過案例：

- 未建立尾款：不通過，應提示尚未建立尾款金流紀錄。
- 尾款尚未入帳：不通過，應提示尾款金流或尾款傳票尚未入帳。
- 訂金尚未入帳：不通過，應提示訂金金流或訂金傳票尚未入帳。

成交前檢查通過後仍應確認：

- 車輛仍為 `保留中`。
- 保留紀錄仍為 `有效`。
- 不新增 Payment Entry、Sales Invoice、Delivery Note 或額外 Stock Entry。

---

## 七、確認成交

確認成交只完成業務狀態轉換，不交車、不出庫、不開銷售發票、不建立收款單、不做收入認列。

通過流程：

```text
保留中車輛
→ 訂金已入帳
→ 尾款已入帳
→ 成交前檢查通過
→ 確認成交
→ 車輛變成已售出
→ 保留單變成已完成
```

不通過案例：

```text
未建立尾款：不通過
尾款尚未入帳：不通過
訂金尚未入帳：不通過
車輛不是保留中：不通過
保留單不是有效：不通過
```

確認成交後應確認沒有新增：

| 文件 | 預期 |
| --- | --- |
| Payment Entry / 收款單 | 不應建立 |
| Sales Invoice / 銷售發票 | 不應建立 |
| Delivery Note / 交車單 | 不應建立 |
| Stock Entry / 庫存異動 | 不應建立 |
| 額外 Journal Entry / 會計傳票 | 不應建立 |

---

## 八、重複尾款防呆

同一筆有效保留紀錄只能建立一筆尾款金流。

若再次建立尾款，系統應阻止，避免重複建立尾款金流與傳票草稿。

---

## 九、通過標準

本輪 QA 通過條件：

- 保留中車輛可建立尾款收款。
- 尾款只建立金流紀錄與傳票草稿。
- 尾款可在會計作業確認入帳並建立正式會計傳票。
- 不建立收款單、銷售發票、交車單或額外入庫單。
- 車輛不會因尾款自動變成已售出。
- 同一筆保留不可重複建立尾款。
- 成交前檢查可正確提示缺漏，且通過時不改車輛與保留狀態。
- 確認成交後車輛為 `已售出`，保留紀錄為 `已完成`。
- 確認成交不新增收款單、銷售發票、交車單、庫存異動或額外會計傳票。
